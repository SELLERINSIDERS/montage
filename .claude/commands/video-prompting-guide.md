# Video Prompting Guide — AI Video Generation Best Practices

Best practices for writing effective AI video generation prompts. Use this when creating image-to-video or text-to-video prompts for VSL scenes, product shots, or any AI-generated video content.

---

## Prompt Structure Formula

Every video prompt should follow this structure:

```
[Shot Type] + [Subject] + [Action] + [Setting] + [Lighting] + [Style] + [Technical]
```

### Example Breakdown

```
"Slow motion close-up of coffee being poured into a white ceramic cup,
steam rising, morning sunlight streaming through window, warm color grading,
cinematic, 4K, shallow depth of field"
```

- **Shot Type**: Slow motion close-up
- **Subject**: Coffee
- **Action**: Being poured
- **Setting**: White ceramic cup, window
- **Lighting**: Morning sunlight
- **Style**: Warm color grading, cinematic
- **Technical**: 4K, shallow depth of field

## Model-Specific Tips

### Google Veo (3, 3.1)
- Excels at realistic, cinematic content
- Supports audio generation (Veo 3+)
- Best with detailed, professional prompts
- Frame interpolation available in 3.1

### Kling (V3 — direct API)
- Excellent image-to-video (animate still images into 5-10s clips)
- **ALWAYS use model `kling-v3`** — `kling-v3-omni` NOT available on direct API (`api.klingai.com`)
- **cfg_scale: `0.4` always** — smoother motion, faithful to source image
- Best at 5-10s per clip — chain clips for longer videos. Minimum 5s (trim shorter in post)
- Aspect ratios: 9:16 (vertical), 16:9, 1:1
- Element referencing NOT available on direct API — character consistency must be baked into source images via anchor blocks
- Audio NOT generated by API — add in post-production (Remotion)

### Seedance
- Strong at dance and human motion
- First-frame control available
- Good for consistent character motion
- Works well with reference images

### Wan 2.5
- Best for image-to-video
- Animates still images naturally
- Good motion prediction
- Works with any image style

### Grok
- Good general-purpose video
- Configurable duration (5-10s)
- Creative interpretations
- Works well with abstract concepts

## Common Mistakes to Avoid

| Mistake | Problem | Better Approach |
|---------|---------|-----------------|
| Too vague | "A nice video" | Specify shot, subject, style |
| Too complex | Multiple scenes | One scene per prompt |
| No motion | Static description | Include camera movement or action |
| Conflicting styles | "Minimalist maximalist" | Choose one aesthetic |
| No lighting | Undefined mood | Specify lighting conditions |
| **Subject-visual mismatch** | Script says "The queen..." but visual shows a harbor | **ALWAYS show the named subject in the first frame** |
| **Video prompt doesn't match image** | Image shows person rubbing neck, video prompt says "neural pathways flickering" | **ALWAYS describe motion for what's actually IN the generated image** |
| **Camera param vs prompt mismatch** | KLING PARAMS says `static` but video prompt says "slow dolly in" | **Camera value and described motion must agree** |
| **Missing image parity** | 48 images but only 42 video prompts | **1 image = 1 video prompt = 1 Kling clip, always** |
| **Ignoring image details** | Image has handmaidens pouring water, video prompt doesn't mention them | **Include all key actions visible in the image** |
| **No character anchor on body parts** | "Elegant feminine hands with gold cuffs" → generates old hands | **ALWAYS add age + ethnicity + skin quality markers** to body-part-only shots |
| **Callback without re-anchor** | "Same setting as Scene 5" → generator doesn't know Scene 5 | **Re-include full character description** in every callback scene |
| **"Film grain" in modern generators** | Gemini/Nano Banana produce artifacts, not cinematic grain | **Use "35mm film aesthetic" instead** — gets the look without artifacts |
| **Boilerplate endings** | Every prompt ends with "Smooth, cinematic movement" | **Vary endings** — repetitive boilerplate = repetitive results |

## Character Consistency & Anchor Blocks (CRITICAL for multi-scene projects)

AI image generators have **NO memory between images**. Each image is generated independently. A character appearing in Scene 1 and Scene 52 will look like two DIFFERENT people unless you force consistency.

### Character Anchor Block

For any recurring character (3+ scenes), create a frozen description block:

```
CHARACTER ANCHOR (copy-paste UNCHANGED into every scene with this character):
"[Age] [ethnicity] [gender], [skin detail], [eye shape/color] with [eye detail],
[hair color/length/style] with [hair accessories], [facial features],
[bearing/posture]. [Clothing]: [specific garments with colors and materials].
[Jewelry/accessories]: [every piece named]. [Consistency note]."
```

### Rules:
1. **Copy-paste unchanged** — never paraphrase or shorten the anchor
2. **Include in EVERY scene** with that character — even callback scenes
3. **Body-part-only scenes need anchor TOO** — hands, forearms, skin close-ups MUST include age, ethnicity, skin quality, and visible jewelry
4. **Wardrobe changes** get a VARIANT anchor (note what changes, what stays)

### Body-Part-Only Scenes (HIGHEST RISK for consistency breaks)

When showing only hands, forearms, or other body parts without a face:
- **WRONG**: "Elegant feminine hands with gold cuffs" → generates random age (25-60)
- **RIGHT**: "Elegant youthful feminine hands of a striking Egyptian queen in her late twenties — taut olive Mediterranean skin, smooth with visible pores. Gold arm cuffs and lapis lazuli bracelets on both wrists"

**Always include in body-part-only prompts**: age marker, skin quality (taut/smooth/youthful), ethnicity marker, visible jewelry that matches the character

## Subject-First Rule (CRITICAL for VSL)

**When the script names a person in the opening line, that person MUST be the first visual frame.** This is non-negotiable for VSL and story-driven video ads.

**Why**: The Fabula Coffee VSL shows Roosevelt the instant his name is spoken — not a coffee plantation, not the White House. The viewer sees a PERSON and hears their name simultaneously, creating instant identification and scroll-stopping power. A landscape or setting shot while a person's name is spoken creates a disconnect that weakens the hook.

**Rule**: When the script says "[Person's name] did [action]," the first visual MUST be that person — their face, their posture, their environment — but THEM, not their world without them.

| Wrong | Right |
|-------|-------|
| Script: "The queen convinced..." → Visual: aerial of a city | Script: "The queen convinced..." → Visual: the queen at a table with maps |
| Script: "Einstein discovered..." → Visual: a university building | Script: "Einstein discovered..." → Visual: Einstein at a chalkboard |
| Script: "Roosevelt drank..." → Visual: coffee beans | Script: "Roosevelt drank..." → Visual: Roosevelt at his desk with a coffee cup |

This rule applies to the FIRST scene of any VSL. After the subject is established visually, subsequent scenes can show locations, objects, and settings freely.

## VSL Scene Prompting (Image-to-Video Pipeline)

For VSL production, the workflow is:
1. **Generate hero image** (Gemini/Midjourney/DALL-E) — one per scene
2. **Animate image** via AI video tool (5-10s per clip) — Kling image-to-video
3. **Mix with real product footage** (shot on phone/camera)
4. **Composite** in editor: clips + captions + music

### Product Placement (MANDATORY for product VSLs)
- **When brand name is first spoken → product MUST be visible** in that scene
- Plan 4 product touchpoints: Brand Introduction → UGC Usage → Product Hero → Lifestyle/CTA
- Tag every product scene with `[PRODUCT REFERENCE: Upload actual photo]` — AI can't invent your product
- UGC-style usage scenes (person taking pill, drinking water) build trust even inside cinematic VSLs
- See Cinematic Director Section 11 for the full 4-Touch Product Placement Framework

### Image Prompt Best Practices (Step 1)
- Include character description for consistency across scenes
- Specify exact color palette for visual cohesion
- Include aspect ratio (9:16 for vertical VSL)
- Describe the FINAL frame composition — AI will generate this as a still

### Video/Motion Prompt Best Practices (Step 2)
- Describe ONLY the motion — the image already defines the visual content
- **Match the prompt to the ACTUAL generated image** — if the image diverges from the script, adapt the video prompt to what's really there
- **Match motion speed to scene energy** — do NOT default to "slow" for everything. Use the motion energy scale below
- Specify duration target (e.g., "5 seconds")
- One camera movement per clip (don't combine pan + zoom + tilt)
- Include atmosphere: "dust particles floating", "steam rising", "hair moving slightly"
- **Include all key subjects/actions visible in the image** — if a handmaiden is pouring water, the person is rubbing their neck, or someone is walking through a field, mention that motion
- **Verify camera direction matches**: if your KLING PARAMS say `static`, don't write "slow dolly in" in the prompt
- **Model**: Always `kling-v3` (NOT v3-omni — not available on direct API)
- **cfg_scale**: Always `0.4`

### Motion Energy Scale (choose per scene)

| Energy | Motion Language | Use For |
|--------|----------------|---------|
| **STATIC** | "Locked shot. No camera movement." | ECU body details, emotional peaks, product confidence |
| **SUBTLE** | "Barely perceptible drift." | Portraits, intimate moments, trust-building |
| **NATURAL** | "Dolly in toward subject." (no speed modifier) | Standard narrative, most scenes — THE DEFAULT |
| **DELIBERATE** | "Measured crane up." "Confident pan right." | Authority, solution reveals, product hero |
| **ENERGETIC** | "Quick zoom in." "Dynamic tracking shot." | Hooks, achievements, montages, pivots |
| **AGGRESSIVE** | "Sharp whip pan." "Rapid push-in." | Pattern breaks, flash cuts, cascades |

**CRITICAL: "Natural" (no speed modifier) is the default.** Only add "slow" when the scene genuinely requires deceleration (CTA close, emotional peaks, trust moments). Only use "fast/quick" when the scene needs energy (hooks, montages, action). Let Kling interpret natural speed — it produces cinematic results without being told "slow."

## Cinematic Video Prompt Standards (Mandatory for VSL)

**Source of truth**: Cinematic Director Section 12 (`.claude/skills/cinematic-director/SKILL.md`). This section summarizes the key rules and adds the atmosphere library.

### The 6 Rules

Every video prompt MUST satisfy all 6:

1. **Camera param matches prompt** — `camera: "pan_right"` = prompt describes rightward pan
2. **Motion matches actual image** — describe what's IN the generated image, not the script
3. **Visible motion elements** — something in the scene physically moves (hair, dust, water, fire, fabric, particles)
4. **Atmosphere layer** — volumetric light, particles, wind, steam, reflections, or caustics
5. **Technical camera grammar** — named lighting techniques, physics directives, lens refs
6. **30-60 words** — tight, no filler, no keyword soup, no boilerplate endings

### Camera Distribution Targets (80+ scene projects)

| Camera Type | Max % |
|-------------|-------|
| static | ≤25% |
| pan (left/right) | 15-25% |
| dolly (in/out) | 12-20% |
| zoom (in/out) | 8-15% |
| tilt (up/down) | 8-12% |
| crane (up/down) | 5-10% |
| orbit (left/right) | 3-8% |

After writing all prompts, count camera types. If static exceeds 25%, reassign scenes.

### Atmosphere Elements Library

Use these to satisfy Rule 4. Pick 1-3 per scene based on setting.

**Palace / Historical:**
- Torch flames dancing, casting shifting shadows on stone walls
- Dust motes drifting through volumetric light beams
- Fabric rippling in warm cross-breeze, linen drapes swaying
- Oil lamp flicker, warm amber light pulsing
- Gold jewelry catching firelight, caustic reflections on skin

**Nature / Landscape:**
- Wind through foliage, grass swaying, branches bending
- Water rippling with oily viscosity, waves lapping at shore
- Heat shimmer distorting distant features, mirage effect
- Mist drifting through valleys, fog rolling across water
- Birdsong implied through rustling leaves, morning dew catching light

**Industrial / Modern:**
- Fluorescent tubes flickering with cold 5600K pulse
- Chemical mist drifting laterally through harsh overhead light
- Conveyor belt rhythm in background, mechanical repetition
- LED indicators pulsing, screen glow on faces
- Sterile air circulation, dust-free but lifeless atmosphere

**Bio-Cinematic (inside the body):**
- Neural sparks firing along axon pathways, bioluminescent pulse
- Muscle fibers contracting and releasing, microscopic tension
- Golden particles flowing through blood vessels, absorption glow
- Cellular membranes rippling, organelles in constant motion
- Synaptic vesicles releasing neurotransmitters, chemical cascade

**Product / Solution:**
- Steam rising from warm bath, water surface catching golden light
- Mineral salt crystals shimmering with subsurface scattering
- Golden hour light shifting across bottle surface, premium bokeh
- Saffron threads swaying in barely perceptible breeze
- Turquoise water bokeh, warm rim light on glass

**CTA / Close:**
- Candle flame swaying gently, warm wax glow
- Page turning slowly, paper texture catching light
- Lake surface with barely perceptible ripple, mirror stillness
- Warm amber light slowly receding, sunset fade
- Dog stretching contentedly, sheets settling, deep breath exhaled

### Bad → Good Examples

| Bad (Generic) | Good (Cinematic) |
|---------------|-----------------|
| "Slow dolly in on the queen. Cinematic. 5 seconds." | "Dolly in toward the queen on her throne. Torch flames dance across gold arm cuffs, casting shifting shadows on limestone walls. Dust motes drift through volumetric light beams. Linen drapes ripple in warm cross-breeze. 5s." |
| "Static shot of Dead Sea. Water moves. 5s." | "Locked wide shot. Mineral-crusted shoreline glistens as hypersaline water laps with oily viscosity. Heat shimmer distorts distant cliffs. Salt crystals catch sunlight in caustic refractions. Warm desert wind lifts fine sand across foreground. 5s." |
| "Pan right across farm. 5s." | "Pan right across industrial greenhouse rows. Fluorescent tubes flicker with cold 5600K pulse. Chemical mist drifts laterally through harsh overhead light. Soil sits inert — no organic motion, no life. 5s." |
| "Zoom in on flower. Beautiful. 5s." | "Zoom in on saffron crocus stigma threads. Morning dew beads catch golden hour light with prismatic refraction. Delicate stamens tremble in barely perceptible breeze. Subsurface scattering through translucent petals creates warm amber glow. 5s." |

## Master Script Alignment Protocol (MANDATORY for VSL projects)

When a master script exists (`vsl/{project_slug}/copy/master_script.md`), all image prompts and video prompts MUST be derived from it. The master script is the **single binding source of truth** — written by the Director using the cinematic-director skill's AUTHOR mode.

### What the Master Script Contains (per scene)

Each scene has sections marked **(BINDING)** — these are contracts you MUST follow:

- **Visual Direction (BINDING)**: Subject, environment, framing, lighting, palette, key details, and exclusions ("What is NOT in frame")
- **Camera Direction (BINDING)**: Camera motion, motion reason, intensity
- **Atmosphere**: Motion elements, sound palette, emotional tone
- **Post-Production Notes**: Edit timing, transitions, callbacks to other scenes

### Image Prompt Alignment — The 7-Point Check

When writing an image prompt for a scene that has a master script entry, you must IMPLEMENT (not re-interpret) the Director's Visual Direction. After writing each image prompt, run this checklist:

| # | Check | What to Compare | Result |
|---|-------|----------------|--------|
| 1 | **Subject match** | Master script "Subject" field → your image prompt's subject | ✓/✗ |
| 2 | **Framing match** | Master script "Framing" (shot size + angle + lens) → your prompt's camera spec | ✓/✗ |
| 3 | **Lighting match** | Master script "Lighting" (named technique) → your prompt's lighting | ✓/✗ |
| 4 | **Palette match** | Master script "Palette" → your prompt's color direction | ✓/✗ |
| 5 | **Key details present** | Every item in master script "Key Details" → appears in your prompt | ✓/✗ |
| 6 | **Exclusions respected** | Every item in "What is NOT in frame" → NOT in your prompt | ✓/✗ |
| 7 | **Scale level match** | Master script "Scale Level" → your prompt's visual approach matches | ✓/✗ |

**Rule: If ANY check is ✗ → fix the prompt before proceeding.** Do not generate images from a prompt that fails alignment.

### Video Prompt Alignment — The 5-Point Check

When writing a video prompt for a scene, you must align with BOTH the master script AND the camera_plan.json (from the Cinematographer skill):

| # | Check | What to Compare | Result |
|---|-------|----------------|--------|
| 1 | **Camera motion matches** | camera_plan.json `camera_type` → your video prompt's described motion | ✓/✗ |
| 2 | **Motion elements from master script** | Master script "Motion Elements" → included in your video prompt | ✓/✗ |
| 3 | **No invented elements** | Nothing in your video prompt that's NOT in the master script or actual image | ✓/✗ |
| 4 | **Matches actual image** | Your video prompt describes motion for what's IN the generated image (Rule 2 from Kling workflow) | ✓/✗ |
| 5 | **Emotional tone consistent** | Master script "Emotional Tone" → your video prompt's energy matches | ✓/✗ |

**Rule: If ANY check is ✗ → fix the prompt before proceeding.**

### scene_prompts.md Output Format (with alignment checks)

When a master script exists, the scene_prompts.md file must include the alignment checks inline:

```markdown
## Scene 02: Roman Warships
**Master Script Ref**: Scene 02 — "Not for gold. Not for land."

### IMAGE PROMPT
[Your image prompt here]

### IMAGE ALIGNMENT CHECK
- Subject match: ✓ Fleet of Roman warships with crimson sails
- Framing match: ✓ Wide shot, low-angle from water level, 24mm
- Lighting match: ✓ Amber sunset backlighting
- Palette match: ✓ Crimson + amber + navy
- Key details: ✓ Bronze ram, eagle emblems, multiple ships
- Exclusions: ✓ No people on deck, no harbor
- Scale level: ✓ MYTHOLOGICAL

### VIDEO PROMPT
[Your video prompt here]

### VIDEO ALIGNMENT CHECK
- Camera motion: ✓ Static (matches camera_plan.json)
- Motion elements: ✓ Ships moving, sails billowing, waves crashing (from master script)
- No invented elements: ✓
- Matches actual image: ✓ [verify after image generation]
- Emotional tone: ✓ Ominous
```

### Common Alignment Failures (What to Watch For)

| Failure | Example | Fix |
|---------|---------|-----|
| **AI adds people** | Master script: "100mm macro of crystals" → prompt includes "a hand lifting" | Check "What is NOT in frame" — add "no hands, no people" to prompt |
| **Wrong camera motion** | Master script: static → video prompt says "slow dolly in" | Read camera_plan.json — use the specified camera_type |
| **Changed metaphor** | Master script: "vintage brass radio dial" → prompt describes "timeline map" | Use master script Key Details verbatim |
| **Added action to callback** | Master script: "Callback to Scene 15, no new action" → prompt adds handmaiden pouring | Callbacks field says "callback" — match the original scene's composition |
| **Scale level mismatch** | Master script: MYTHOLOGICAL (hundreds of people) → prompt shows 2 people | Scale Level constrains scope — MYTHOLOGICAL means EPIC |

## Video Re-alignment Protocol (After Image Revisions)

When images go through multiple revision rounds (V1 → V2 → selective regenerations → final 2K), the video prompts in `scene_prompts.md` become stale — they were written for the ORIGINAL images, not the revised ones. This protocol ensures video prompts match the ACTUAL final images before Kling generation.

### When to Run Re-alignment

Run this after ALL image revision rounds are complete (no more changes expected). Produces `scene_prompts_final.md` — the sole input for building the Kling manifest.

### Step 1: Categorize Each Scene

| Category | Criteria | Action |
|----------|----------|--------|
| **Unchanged** | Final image = original V1 image | Copy video prompt verbatim |
| **Minor revision** | Same composition, adjusted details | Surgical update (fix specific elements only) |
| **Major revision** | New composition or framing | Full rewrite using the 5-Point Check |
| **Complete replacement** | Entirely new concept | Full rewrite + verify camera_plan feasibility |

### Step 2: Re-align Changed Scenes

For each revised scene:

1. **Examine the final image** — note what's actually in frame
2. **Compare against original video prompt** — identify mismatches (wrong subject, missing elements, describing things no longer in the image)
3. **Check camera_plan feasibility** — does the planned camera motion still work? (e.g., orbit on a flat macro composition = infeasible → flag)
4. **Rewrite the video prompt** describing motion for the ACTUAL final image
5. **Run the 5-Point Video Alignment Check**:
   - Camera motion matches camera_plan.json? ✓/✗
   - Motion elements from master script? ✓/✗
   - No invented elements? ✓/✗
   - Matches ACTUAL final image? ✓/✗
   - Emotional tone consistent? ✓/✗

### Step 3: Produce scene_prompts_final.md

Output format — same as `scene_prompts.md` but with:
- A **change log** at the top listing revised scenes and why
- ALL scenes included (unchanged copied verbatim + revised ones updated)
- Updated alignment checks for revised scenes
- **Parity count** at the bottom (total video prompts must equal total images)

### Key Rules

- **Don't touch unchanged scenes** — if the image didn't change, the video prompt is still valid. Don't "improve" it
- **Camera plan values preserved unless infeasible** — a new composition may make the original camera motion nonsensical. Flag and adjust
- **Master script still binding** — re-aligned prompts must still respect the Director's emotional intent and atmosphere
- **scene_prompts_final.md replaces scene_prompts.md** for all downstream use (Kling manifest, parity checks, audio design)
